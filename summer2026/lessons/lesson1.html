🐣 День 1: Введение в Python

Переменные, типы данных, вывод информации, условный оператор, циклы

Теория

Python — язык с простым синтаксисом. Переменные создаются автоматически при присваивании:

name = "Артём"
age = 12
print("Привет, меня зовут", name)
print("Мне", age, "лет")

Основные типы данных:

  • str — строка (текст в кавычках)
  • int — целое число
  • float — дробное число

Условный оператор: if, elif, else

Условный оператор позволяет программе принимать решения. В Python он записывается с помощью ключевых слов if, elif (else if) и else.

if x > 0: print("Положительное") elif x == 0: print("Ноль") else: print("Отрицательное")

Логические операторы

Позволяют комбинировать несколько условий:

  • and — оба условия истинны
  • or — хотя бы одно условие истинно
  • not — отрицание условия
if age >= 18 and has_passport: print("Доступ разрешён")
Важно! После if, elif и else обязательно ставится двоеточие :, а тело условия записывается с отступом (обычно 4 пробела).

Python не использует фигурные скобки — отступы определяют структуру кода!

Интересный факт

Условный оператор существует почти во всех языках программирования, но в Python его синтаксис считается одним из самых читаемых и элегантных. Это делает код похожим на обычный английский текст!

Советы по решению задач

  • Всегда проверяйте граничные значения (например, 0, отрицательные числа).
  • Используйте elif, а не вложенные if, если условия взаимоисключающие.
  • Для сравнения строк учитывайте регистр: "Ананас" != "ананас".

Пример: Проверка возраста

age = int(input("Сколько вам лет? ")) if age < 12: print("Вам нельзя на этот фильм!") else: print("Добро пожаловать!")

Цикл while

Цикл while выполняет блок кода пока условие истинно. Как только условие становится ложным — цикл останавливается.

count = 0 while count < 3: print("Привет!") count = count + 1

Выведет «Привет!» три раза. Обратите внимание: переменная count обязательно должна изменяться внутри цикла!

⚠️ Осторожно! Если забыть изменить переменную в теле цикла, получится бесконечный цикл — программа зависнет!

while True:
    print("Помогите!") # Никогда не остановится

🐍 Интересный факт

Цикл while — одна из самых древних управляющих конструкций в программировании. Он позволяет реализовать любую логику повторения, даже если заранее неизвестно, сколько раз нужно повторить действие.

💡 Советы по циклу while

  • Всегда проверяйте, что условие цикла рано или поздно станет False.
  • Используйте break, чтобы досрочно выйти из цикла (например, при вводе «стоп»).
  • Используйте continue, чтобы пропустить остаток текущей итерации и перейти к проверке условия.

Задание 1: Система ранжирования киберспортсменов

Легенда задачи

Ты — junior-аналитик профессиональной киберспортивной команды. Перед важным турниром тренер проводит серию тренировочных матчей и собирает статистику игроков с помощью специальных игровых датчиков.

Иногда датчики сбоят и выдают некорректные значения (например, отрицательные числа или баллы >100). Твоя задача — написать логический блок программы, который автоматически отфильтрует сбои оборудования и присвоит игроку честный ранг на основе набранных очков.

Напиши программу, которая проверяет очки и определяет статус:

  • Если число score вне диапазона 0 – 100 → 🔴 Датчик сбоит. Игра не засчитывается.
  • 90 – 100 очков → 🥇 Ранг: LEGEND
  • 75 – 89 очков → 🥈 Ранг: PRO
  • 50 – 74 очков → 🥉 Ранг: AMATEUR
  • 0 – 49 очков → 📍 Ранг: BEGINNER

Итог: В конце сессии программа выводит финальный отчёт: сколько всего матчей было успешно обработано без сбоя датчиков.

Задание 2: Распределяющая Шляпа

Шляпа анализирует два аспекта: главное качество и вдохновение. Она ищет совпадения, исключает противоречия и принимает решение по комбинации ответов.

Правила распределения

Факультет Условие (логика)
Гриффиндор (качество == "храбрость" or вдохновение == "подвиги") and вдохновение != "тайны"
Слизерин (качество == "хитрость" or качество == "амбиции") and (вдохновение == "победа" or вдохновение == "тайны")
Когтевран (качество == "ум" or вдохновение == "знания") and качество != "амбиции"
Пуффендуй качество == "честность" or качество == "доброта" or вдохновение == "помощь другим"
Иначе "Шляпа в замешательстве... Попробуй ещё раз"

Твоя задача

Напиши программу, которая:

  1. Спрашивает: "Какое качество ты считаешь своим главным?"
  2. Спрашивает: "Что тебя вдохновляет больше всего?"
  3. Определяет факультет по таблице выше (используй if-elif-else, and, or, != и скобки)
  4. Выводит результат

📐 Математические условия

ФигураМатематическое условиеРезультат при ошибке

⏳ Задание 4: Парадокс временной петли

Сюжет: Машина времени работает по странному алгоритму ИИ. Она переносит человека в прошлое, только если введенный год является «Зеркально-високосным».

Математическая суть: Сложная булева логика, остатки от деления, палиндромы.

Задача

На вход подается четырехзначный год (например, 2024). Программа должна проверить:

  • Год должен быть високосным: делится на 4, но не делится на 100, ЛИБО делится на 400.
  • И при этом сумма первых двух цифр года должна быть равна сумме двух последних цифр (например, для 2024: 2+0 ≠ 2+4, не подходит).

Вывод

  • Если оба условия сошлись → «Портал открыт, добро пожаловать в прошлое!»
  • Если год просто високосный → «Времени недостаточно для прыжка»
  • В остальных случаях → «Машина времени превратилась в тыкву»

🎒 Задание 5: Бездонный рюкзак перед походом

Сюжет: Вы собираетесь в поход и пытаетесь запихнуть вещи в рюкзак. Но его объём не бесконечен. Вы кидаете вещи одну за другой, пока рюкзак не начнёт трещать по швам.

Как работает while: Цикл работает, пока свободное место в рюкзаке больше нуля.

Задача

  • Программа задаёт стартовую вместимость рюкзака (например, 20 кг).
  • В цикле спрашивает: «Что вы хотите положить?» и «Сколько это весит?».
  • Вычитает вес из оставшейся вместимости.
  • Если вес вещи больше, чем осталось места → пишет «Это уже не влезет! Попробуй что-то полегче» и не уменьшает место.
  • Цикл завершается, когда:
    • свободное место стало 0 кг (идеальная упаковка), ИЛИ
    • пользователь ввёл стоп-слово "выход".

Вывод

Список вещей (если хранишь их в строке) или финальное сообщение: «Рюкзак собран! Свободного места: Х кг».

⛏️ Задание 6: Побег из заброшенной шахты

Сюжет: Персонаж бежит по тёмной шахте, за ним гонятся зомби. Нужно пробежать 100 метров до выхода. Каждую секунду выбирается режим бега.

Как работает while: Цикл работает, пока дистанция < 100 и здоровье > 0.

⚙️ Числа и механика

  • Старт: дистанция = 0 м, зомби сзади = 30 м, здоровье = 100, выносливость = 50
  • 1. Спринт: +12 м вперёд, −25 выносливости, зомби приближаются на +8 м
  • 2. Шаг: +5 м вперёд, +15 выносливости (макс. 50), зомби приближаются на +15 м
  • Атака зомби: если расстояние до зомби ≤ 0 → −20 здоровья, зомби отбрасываются на 5 м назад
  • Победа: дистанция ≥ 100 м
  • Поражение: здоровье ≤ 0

🎯 Твоя задача

Напиши цикл while, который каждый ход запрашивает выбор (1 или 2), обновляет переменные по правилам выше, выводит текущее состояние и проверяет условия победы/поражения.

📤 Пример вывода

Вы на 45-м метре. До зомби: 12 м. HP: 80 | Выносливость: 30
Ура, вы выбрались на свет! или Зомби оказались быстрее... Игра окончена.

💳 Задание 7: Валидация банковской карты

Сюжет: Вы — ведущий инженер кибербезопасности банка. При ручном вводе 16-значного номера карты пользователи часто допускают опечатки. Пересылать неверный номер на сервер долго и небезопасно. Ваша задача — написать локальный скрипт проверки.

Алгоритм Луна (Luhn): стандарт, используемый Visa, Mastercard и МИР по всему миру.

📐 Правила проверки

  1. Убедись, что длина = 16.
  2. Пройди по цифрам справа налево.
  3. Чётные по позиции цифры: просто прибавь к общей сумме.
  4. Нечётные по позиции цифры: умножь на 2. Если результат > 9, вычти 9.
  5. Сложи все полученные значения. Если итоговая сумма кратна 10 → номер валиден.

Пример валидного номера: 4276 4400 1336 1511

Твоя задача

Напиши программу, которая принимает номер карты, применяет алгоритм Луна и выводит «✅ Карта валидна» или «❌ Ошибка в номере карты».

🌌 Задание 8: Числа Армстронга и цифровые монстры

Сюжет: В НАСА ищут аномальные числа для кодирования сигналов. Одно из таких явлений — числа Армстронга (числа, сумма цифр которых, возведённых в степень количества цифр, равна самому числу. Например, 153 = 1³ + 5³ + 3³).

Как устроена сложность: Работа с разрядами числа без использования строк. Сначала нужно в цикле посчитать количество цифр, а затем во втором цикле возвести каждую цифру в эту степень.

📐 Алгоритм проверки

  1. Пользователь вводит целое число. Сохрани его копию для финальной проверки.
  2. Первый цикл while: считай количество цифр, деля число на 10, пока оно не станет 0.
  3. Второй цикл while: извлекай каждую цифру (через остаток `% 10`), возводи её в степень (количество цифр) и прибавляй к сумме.
  4. Проверка if: если итоговая сумма равна исходному числу → это число Армстронга.

Твоя задача

Реализуй оба цикла строго через арифметические операции (//, %), не переводя число в строку. Выведи вердикт: «Это число Армстронга!» или «Это обычное число».

🚌 Задание 9: Ближайшая автобусная остановка

Сюжет: Автобусные остановки расположены через каждые K метров вдоль проспекта (на расстояниях 0, K, 2K, 3K и т.д.). Маша прошла N метров от начала и хочет сесть в автобус. Как далеко ей идти до ближайшей остановки?

📐 Формат данных

  • Вход: два целых числа K и N (в отдельных строках). 1 ≤ K, N ≤ 2·10⁹.
  • Выход: одно целое число — минимальное расстояние до остановки.

Пример

K = 600, N = 2000 → Остановки: 0, 600, 1200, 1800, 2400...
Маша на 2000 м. Ближайшая — 1800 м. Расстояние: 200.

🧗 Задание 10: Надёжная страховка альпиниста

Сюжет: Альпинист пристегивается тремя карабинами, которые выдерживают массу a, b и c кг. Страховка надёжна, если хотя бы два карабина выдерживают массу альпиниста со снаряжением. Определите максимальную массу, для которой страховка будет надёжной.

📐 Формат данных

  • Вход: три натуральных числа a, b, c (в отдельных строках). 1 ≤ a, b, c ≤ 2·10⁹.
  • Выход: одно целое число — максимальная надёжная масса.

Пример

Вход: 28, 15, 10 → Выход: 15

🕵️ Задание 11: Дедуктивная игра «Бейглз»

Сюжет: В логической игре «Бейглз» необходимо по подсказкам угадать секретное число из трёх цифр. Компьютер загадывает число, а ты пробуешь его отгадать за 10 попыток.

Правила подсказок

  • Fermi — правильная цифра стоит на правильном месте
  • Pico — правильная цифра, но на неправильном месте
  • Bagels — в догадке нет ни одной правильной цифры

Примечание: цифры в секретном числе не повторяются.

Твоя задача

Реализуй игровой цикл: компьютер загадывает число, игрок вводит догадку, программа анализирует совпадения и выдаёт подсказки. Игра заканчивается при полном угадывании или после 10 попыток.

Алгоритм проверки

  1. Сравни догадку и секретное число по позициям → собирай Fermi
  2. Для остальных цифр проверяй наличие в секретном числе → собирай Pico
  3. Если списки подсказок пусты → выводи Bagels
  4. Иначе выводи все подсказки через запятую (например: Fermi, Pico)
